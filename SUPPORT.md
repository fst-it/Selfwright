# Getting help with Selfwright

## Bug reports

Open a [GitHub Issue](https://github.com/fst-it/Selfwright/issues) for confirmed bugs.
Include the command you ran, the error output, and your Node/pnpm versions.

## Questions and discussion

Use [GitHub Discussions](https://github.com/fst-it/Selfwright/discussions) for:

- questions about setup or configuration
- asking how a concept works (truth layer, archetypes, drifts, fitness functions)
- sharing what you built with the framework
- proposing ideas before opening a feature request issue

## Documentation

- **[docs/MANUAL.md](docs/MANUAL.md)** — the operating manual; structured for lookup
- **[docs/use-cases.md](docs/use-cases.md)** — worked examples for scoring, tailoring, and scanning
- **[docs/fitness-functions.md](docs/fitness-functions.md)** — the full catalog of architectural checks
- **[docs/adr/](docs/adr/)** — architecture decision records; the why behind every major design choice

## Security issues

Do not open a public issue for a security vulnerability. Report privately via a GitHub
security advisory instead. See [SECURITY.md](SECURITY.md) for the full coordinated-disclosure
process.

## What is not supported here

Selfwright is the open-core framework. The private data layer — your truth files, applications,
drifts, and compensation data — is yours and lives in your own `Selfwright-data` repository.
Questions about personal career decisions, confidential application strategy, or the contents of
your private data repo are not appropriate for public issues or discussions.
